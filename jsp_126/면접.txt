4) JDBC(Java DataBase Connectivity) : 자바에서 데이터베이스와 관련된 작업을 처리할 때 사용하는 API
     오라클 JDBC 드라이버 : DBMS와의 통신을 담당하는 자바 클래스이며 jar 파일형태(ojdbc6.jar)로 제공 


5) 커넥션풀
			: 데이터베이스와 연결된 커넥션을 미리 만들어서 풀(pool)속에 저장해두고 있다가 
			    필요할 때 커넥션을 풀에서 가져다 쓰고 다시 풀에 반환하는 기법
			- 풀 속에 미리 커넥션이 생성되어 있기 때문에 커넥션을 생성하는데 드는 연결시간을 줄일 수 있다.
			- 커넥션을 계속해서 재사용하기 때문에 생성되는 커넥션 수가 일정하게 유지된다
		==> 이번 프로젝트에서는 DBCP(DataBase Connection Pool) 방식을 이용
		==> DBCP(DataBase Connection Pool) : DB와 커넥션을 맺고 있는 객체를 관리하는 방식

6) MVC 패턴 정의
 모델 2 구조는 모델 1 구조와 달리 웹 브라우저의 요청을 하나의 서블릿에서 처리한다.
 
 >> MODEL		:	비즈니스 영역의 로직을 처리한다
					*  로직클래스 = 서비스 + DAO
					*  자바빈 (= DTO) 
 >> VIEW 		:	비즈니스 영역에  대한 뷰(즉 사용자가 보게 될 결과 화면) ~ 브라우저
 					*  JSP
 >> CONTROLLER	:	사용자의 요청 처리와 흐름제어를 담당한다
					*  서블릿
					- 버튼 클릭, 메뉴 클릭, 링크 클릭, url입력
MVC 약자 및 개념 정리
  예) 회원가입 흐름도 : Controller -> 서비스 -> DAO(DB처리) -> 서비스(jsp에게 결과전달 ) -> jsp(결과출력)
